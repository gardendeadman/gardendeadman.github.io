블로그 자립하기

2020-01-28 16:01

개요

나는 예전에 인터넷에서 무수히 많은 개인홈페이지란 것들이 운영되고 있는 것을 봤다. 당시에는 애니메이션 같은 것에 흥미가 있었는데, 그런 주제만으로도 정말 다양한 개인홈페이지들을 볼 수가 있었다.
그래서 나도 언젠가 그런 걸 만들어 보면 좋겠다고 생각했는데 어느덧 시간이 흘러서 최근에야 ‘정적 사이트 생성기’란 게 있다는 걸 알게 됐다.
이렇게 내가 ‘블로그 자립’을 위해서 준비한 과정을 정리해보려고 한다.
이번에야말로 -예전 기억을 떠올리며- 인터넷 상에 내 관심사를 정리하기 위한 공간을 만들어보자!

순서

  1. Github Pages
  2. GoormIDE
  3. Hexo
  4. Disqus
  5. Google AdSense
  6. SEO
  7. Google Analytics
  8. Hexo Migration

1. Github Pages

시작은 묻지도 따지지도 말고 Github Pages로 가자.
Github는 통상적인 의미에서의 서버란 게 없이도 우리의 컨텐츠와 블로그 그 자체를 서버에 -남의 것이지만 결국은 서버- 올려둘 수 있는 서비스이다.
본래 이 서비스는 프로그래머들이 개발하는 소프트웨어를 버전별로 관리할 수 있도록 하는 것으로, -유식한 말로 ‘형상 관리’라고 한다- 우리의 블로그도 마치 소프트웨어처럼 버전별로 관리될 것이므로 이 서비스에 적합하다.
따라서 Github에 계정을 만들고, Repository라고 하는 것 -말 그대로 저장소- 을 생성하도록 하자.

pages.github.com

2. GoormIDE

ide.goorm.io

구름IDE는 설치가 필요없는 클라우드 통합 개발환경으로
팀의 개발 효율성을 극대화하는데 도움을 줍니다.

그래서 난 어느 곳에서든 포스팅 할 수 있도록 구름IDE를 선택했다.

3. Hexo

hexo.io

“Hexo는 블로그 프레임워크입니다.”

이제 Get started에 있는 그대로 Hexo를 설치해보자.

여기까지하면 벌써 기본적인 형태의 ‘정적인’ 블로그가 생성된다.

4. Disqus

disqus.com

Disqus는 웹상의 수많은 사이트에서 사용되는 네트워크 커뮤니티 플랫폼으로, Hexo로 생성된 정적페이지에서 댓글시스템을 사용할 수 있도록 해줍니다.

5. AdSense

google.com/adsense

이번에는 수익에 보탬이 되는 구글 애드센스를 설치해보자.

6. SEO

search.google.com/search-console

SEO는 검색 엔진 최적화의 약자이다.
그런데 검색 엔진 최적화는 왜 하는 걸까?
우선 SEO를 위해서 이미 만들어진 Hexo의 플러그인 몇 가지를 설치해보자.

7. Analytics

analytics.google.com/analytics/web

8. Hexo Migration

사실 Hexo만이 유일한 블로그 프레임워크는 아니다.
따라서 향후에 다른 프레임워크로 갈아타려면 마이그레이션을 할 수 있어야 하는데, 앞서 기술한 내용에 비해 필수적인 요소는 아니기에 단지 그런 가능성이 있다는 점을 인지하는 수준에서 만족하자. 사실은 귀찮다.
